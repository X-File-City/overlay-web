# Overlay Website Flow & Content Strategy

## App Theme

**Overlay** is a macOS application that serves as your **personal, unified AI interaction layer**. The core value proposition:

> *"Everything you need from AI—chats, notes, browser, agents—powered by your voice, all in one place."*

The app eliminates context switching by providing instant-access overlays for voice, notes, chats, browser, and agents—all available through simple keyboard shortcuts without leaving the current task.

---

## Website Flow Overview

The landing page uses a **scroll-driven narrative** with 11 sequential sections. Each section is fixed-position with opacity transitions tied to scroll progress, creating a cinematic presentation that mirrors the app's "flow without friction" ethos.

**New Messaging Hierarchy:**
1. Position overlay as an AI layer (not just "reimagined computing")
2. Lead with value ("everything you need from AI")
3. List capabilities upfront, then demonstrate each
4. Conclude by naming the paradigm and circling back to "personal computing, reimagined"

## Section-by-Section Breakdown

### 1. Hero Section (Scroll: 0% - 8%)
**Purpose:** Immediate brand recognition + AI positioning

**Content:**
- Animated logo (scales down on scroll)
- Title: "overlay" (serif typography)
- Tagline: "your personal, unified ai interaction layer"
- Download button with Apple icon: "download for mac"
- Demo link to X/Twitter post

**Theme Connection:** 
The new tagline immediately positions overlay as an **AI tool**—a layer that sits between you and AI. "Personal" and "unified" emphasize it's yours and brings everything together.

---

### 2. Value Proposition Section (Scroll: 8% - 14%)
**Purpose:** State the core value simply

**Content:**
- Single statement: "everything you need from AI"
- Large serif typography, centered

**Scroll Animation:**
- Fades in at 8%, holds, fades out at 14%

**Theme Connection:**
Bold, confident claim. No explanation needed—sets up anticipation for what "everything" means.

---

### 3. Feature List Section (Scroll: 14% - 22%)
**Purpose:** Preview all capabilities before diving deep

**Content:**
- Line 1: "chats, notes, browser, agents"
- Line 2 (fades in below): "powered by your voice"

**Scroll Animation:**
- 14-16%: Feature list fades in

**Theme Connection:**
Lists the "what" (features) then reveals the "how" (voice). Voice is positioned as the **interaction method** that powers everything, not just another feature.

---

### 4. All In One Place Section (Scroll: 22% - 32%)
**Purpose:** Show unified control before individual features

**Content:**

**Scroll Animation:**
- 24%: Pill appears and expands
- 26%: Voice transcription appears (bottom)
- 27%: Notes overlay slides in (left)
- 28%: Chats overlay drops (top)
- 29%: Browser overlay slides (right)
- 30%: Agents indicator appears

**Theme Connection:**
Moved earlier to establish the unified concept before showing individual features. User sees the "one place" promise fulfilled, then gets to explore each piece.

## Visual Design System

### Colors
- **Background:** `#fafafa` (warm off-white)
- **Primary Text:** `#0a0a0a` (near-black)
- **Secondary Text:** `#71717a` (muted gray)
- **Tertiary Text:** `#a1a1aa` (light gray)
- **Accent:** `#007aff` (blue for interactive states)

### Typography
- **Headlines:** Serif font (elegant, editorial feel)
- **Body/UI:** System sans-serif (clean, readable)
- **Scale:** Large headlines (4xl to 6xl) create visual impact

### Visual Motifs
- **Pill shapes:** Rounded rectangles (12-28px radius) represent the overlay aesthetic—soft, approachable, non-intrusive
- **Transparency:** Glass-morphism effects suggest overlays floating above content
- **Waveforms:** Visual shorthand for voice/audio interaction
- **Minimal chrome:** Few borders, maximum whitespace

## Interactive Philosophy

The website **embodies** the app it describes:

1. **No page navigation** — Single continuous experience (like staying in one app)
2. **Scroll as progression** — Linear flow without jumps or reloads
3. **Reveals on demand** — Content appears as you "need" it through scrolling
4. **Minimal UI** — No navigation bars, sidebars, or distracting elements
5. **Immediate feedback** — Interactive demos respond instantly

The landing page is itself an **overlay experience**—floating above the traditional website paradigm, preserving the visitor's attention flow while delivering information.
